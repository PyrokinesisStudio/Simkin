Simkin Built-in Functions

The following functions are available in all Simkin scripts.

There are comparatively few of these - applications embedding Simkin should provide a great deal more!

String handling

  • length(<value>)
    this returns the length of the parameter (converted to a string)
    For example: 
    a="1.0";
trace(length(a)); // will output "3"
  • charAt(<value>,<pos>)
    this returns the the character at position pos within the parameter value.
    pos starts at 0 for the first character.
    For example: 
    a="1.0";
trace(charAt(a,2)); // will output "0"

Debugging Aids

  • trace(<value>)
    this outputs a message to the tracer (as specified by the underlying application)
    For example: 
    a="1.0";
trace(a); // will output "1.0" to the application tracer

Type Coercion Functions

  • toInt(<value>)
    this returns a version of the parameter converted to an integer.
    For example: 
    a="1.0";
trace(toInt(a)); // will output "1"
  • toFloat(<value>)
    this returns a version of the parameter converted to a floating point value.
    For example: 
    a="1";
trace(toFloat(a)); // will output "1.0000"
  • toString(<value>)
    this returns a version of the parameter converted to a string.
    For example: 
    a="1.0";
trace(toString(a)); // will output "1.0"
  • toChar(<value>)
    this returns a version of the parameter converted to a character.
    For example: 
    a="1.0";
trace(toChar(a)); // will output "1"
  • toBool(<value>)
    this returns a version of the parameter converted to a boolean value.
    For example: 
    a="1.0";
trace(toBool(a)); // will output "true"
  • isObject(<value>)
    this returns true if the parameter is an object (as opposed to a number, string, character or boolean)
    For example: 
    a="1.0";
trace(isObject(a)); // will output "false" 
trace(isObject(self)); // will output "true"
